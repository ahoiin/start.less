start.less

Start.less is a lightweight (14kb) style library for web apps.
It is perfect to start with a webapp or just create a fast app-prototype.
Start.less is based on a responsive layout, you can use several offered webfonts for your icons and customize your app easily with the included .less files.

Download start.less on GitHub or include it via http://lib.appgestalt.com/startless/start.css.
Check out the Starter Template.

Forms

Default styles

Individual form controls receive styling, but without any required base class on the <form> or large changes in markup. Results in stacked, left-aligned labels on top of form controls.

Example block-level help text here. 
<form>
  <fieldset>
    <label>Label name</label>
    <input type="text" placeholder="Type something…">
    <span class="help">Example block-level help text here.</span>
    <label class="checkbox">
      <input type="checkbox"> Check me out
    </label>
    <button type="submit">Submit</button>
  </fieldset>
</form>

Optional layouts

Included with Start are two optional form layouts for common use cases.

Horizontal form

Right align labels and float them to the left to make them appear on the same line as controls. Requires the most markup changes from a default form:

Add .horizontal to the form, Wrap labels and controls in .group, Wrap any associated controls in .controls for proper alignment. 
<form class="horizontal">
  <div class="group">
    <label for="inputEmail">Email</label>
    <div class="controls">
      <input type="text" id="inputEmail" placeholder="Email">
    </div>
  </div>
  <div class="group">
    <label for="inputPassword">Password</label>
    <div class="controls">
      <input type="password" id="inputPassword" placeholder="Password">
    </div>
  </div>
  <div class="group">
    <div class="controls">
      <label class="checkbox">
        <input type="checkbox"> Remember me
      </label>
      <button type="submit">Sign in</button>
    </div>
  </div>
</form>

Supported form controls

Examples of standard form controls supported in an example form layout.

Inputs

Most common form control, text-based input fields. Includes support for all HTML5 types: text, password, datetime, datetime-local, date, month, time, week, number, email, url, search, tel, and color.

Requires the use of a specified type at all times.

<input type="text" placeholder="Text input">

Textarea

Form control which supports multiple lines of text. Change rows attribute as necessary.

<textarea rows="3" placeholder="Textarea field..."></textarea>

How to use

Webfonts on server

You have to add the following code to your .htaccess file to import webfonts without any problems. 

AddType font/ttf .ttf
AddType font/eot .eot
AddType font/otf .otf
AddType font/woff .woff

Select a webfont

Default the webfont Entypo by Daniel Bruce is used.
To change the webfont, please open icons.less and change the following variables. It is recommended to store in these folder the .woff, .ttf and .eot file for the best browser support.

// Create Webfonts
// --------------------------------------------------
// Set the name of webfont, the font-file should have the same name
@webfont:	'entypo'; 
// Path to Webfont, Including .woff, .ttf and .eot files 		
@iconWebfontPath:'fonts/entypo/fonts/';

Multiple webfonts

If you want to include more than one font on a site, you can simply chain a class to it that will apply the appropriate font-family.

[class*="entypo icon-"]:before {
  font-family: "entypo";  
}

How to use in html

All icons can be accessed by two ways. You can use data-icon="CHAR" with any kind of tag or <icon>CHAR</icon>. If you are using more than one webfont-iconset, you should use an unique class and add it to the tag, as well.


# 
<span data-icon="#"></span>
<icon>#</icon>

The other option is to add a class, prefixed with icon-, to any kind of tag. See the character maps of the webfonts for the exact class.

<span class="icon-iw"></span>

Layout

Responsive Layout

As the default, the layout adapts itself to the screensize and the following items: header, nav section, section, article, footer section, .container get a specific width. However, if you want to use the full width, just use the class .full-width.

<!DOCTYPE html>
<html lang="en">
<body>
  <nav>...</nav>
  <article>...</article>
  <footer>...</footer>
</body>
</html>

Centered Layout

Due to the automatic width of all important elements, the whole site can easily be centered by using the class .center. The class can also combined with the grid system.

<!DOCTYPE html>
<html lang="en">
<body>
  <article class="center">...</article>
</body>
</html>

The navbar and footer should be include an element which has the class .center so the whole footer and navbar can use the whole width and only the content is centered.

<!DOCTYPE html>
<html lang="en">
<body>
  <nav>
    <section class="center">...</section>
  </nav>
  <article class="center">...</article>
  <footer>
    <section class="center">...</section>
  </footer>
</body>
</html>
